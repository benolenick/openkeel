Key Design Decisions

Intent matching, not keyword search. "make text bigger" should find Display Scaling, Font Size, AND Accessibility Zoom — all three, ranked by likelihood. The LLM understands what you mean, not just what you typed.
One-click apply. Don't just show where the setting is — offer to change it right there. "Adjust →" opens an inline slider/toggle. No navigation needed.
Super+S global hotkey. Available from anywhere, even inside apps. Like Spotlight but specifically for system settings.

Your settings survive updates, reboots, and distro upgrades — always

Key Design Decisions

Drift detection, not just backup. The system watches for changes and tells you WHO changed it (which update, which package). Users don't just want their setting back — they want to know why it moved.
"Always protect this" creates a contract. Once locked, LLMOS will reapply this setting after any event — update, reboot, package install. The user should never have to set it twice.
Show the diff. Users with technical skill want to see exactly what changed. Novice users just want the "Restore" button. Both are served by the same card.

Key Design Decisions

Track from install, not just at uninstall. LLMOS monitors file creation per-app from the moment it's installed (via eBPF/fanotify). At uninstall time, it already knows every file that app ever wrote.
Smart defaults for user data. Game saves, documents, and projects are flagged separately from app junk. "Remove everything" is always available but never the default — data loss is the thing people fear most.
Residual scan for already-installed apps. For apps installed before LLMOS, run a one-time scan that maps known dotfile patterns to installed packages. Shows residual file size on the app card so users can see the mess.
